# Rust Wipe Hub

Многостраничный (SPA-стиль, вкладки без перезагрузки) фанатский справочник по **Rust**:
живой счётчик до force wipe, гайды по биомам и ресурсам, монументы, 3D-карта острова
и калькуляторы рейда/апкипа.

Тёмная индустриальная тема (металл / ржавчина, акцент — рыжий-оранжевый + огонь),
собранная строго по дизайн-системе из скилла **ui-ux-pro-max**.

## Вкладки

1. **Вайп** — живой таймер до force wipe (первый четверг месяца, 19:00 UTC), дата в вашем
   часовом поясе и в UTC, таблица ближайших 6 вайпов, состояние `WIPE IS LIVE` + автопересчёт.
2. **Ресурсы и биомы** — карточки биомов и таблица ресурсов (где и как фармить).
3. **Монументы** — карточки с уровнем сложности (tier), радиацией, нужными картами доступа
   и лутом; фильтр по сложности.
4. **3D-карта** — стилизованный 3D-рельеф острова на Three.js (вращение + зум, OrbitControls).
   Заготовлено место под реальную карту по сиду через RustMaps API.
5. **Калькуляторы** — рейд (сера/взрывчатка на постройки) и апкип (расход на TC).

## Технологии

- Статичный сайт: **HTML + CSS + чистый JavaScript** (без сборщиков).
- **Three.js** (через CDN + importmap) — только для вкладки 3D-карты, грузится лениво.
- Шрифты Google Fonts: **Orbitron** (заголовки) + **JetBrains Mono** (текст/данные).
- Адаптивность: 375 / 768 / 1024 / 1440px. Учтён `prefers-reduced-motion`, доступность
  (клавиатурная навигация по вкладкам, видимый фокус, контраст ≥ 4.5:1).

## Структура файлов

```
.
├── index.html                       # разметка, вкладки, importmap для Three.js
├── style.css                        # все дизайн-токены (CSS-переменные) и стили
├── script.js                        # логика: вкладки, таймер, рендер данных, 3D, калькуляторы
├── data.js                          # ИГРОВЫЕ ДАННЫЕ (монументы, биомы, ресурсы) — правьте здесь
├── README.md
└── design-system/
    └── rust-wipe-hub/
        └── MASTER.md                # дизайн-система (источник правды), сгенерирована ui-ux-pro-max
```

> **Где править данные:**
> - Монументы / биомы / ресурсы → `data.js` (объекты `MONUMENTS`, `BIOMES`, `RESOURCES`).
> - Цифры калькуляторов → `script.js`, объекты `RAID_DATA` и `UPKEEP_DATA`.
> - Час вайпа → `script.js`, константа `WIPE_HOUR_UTC` (по умолчанию `19`).

## Как запустить локально

Так как `script.js` — ES-модуль, открывать `index.html` двойным кликом (`file://`)
нельзя — нужен локальный сервер:

```bash
# Python 3 (есть почти везде)
python3 -m http.server 8080
# затем открыть http://localhost:8080
```

Альтернативы: `npx serve` или расширение **Live Server** в VS Code.

> Вкладка **3D-карта** и веб-шрифты требуют интернета (грузятся с CDN). Если Three.js
> не загрузился — сайт продолжит работать, а на вкладке карты покажется понятное сообщение.

## Как опубликовать бесплатно на GitHub Pages

1. Запушьте проект в репозиторий GitHub (файлы должны лежать в корне).
2. В репозитории: **Settings → Pages**.
3. **Source** → `Deploy from a branch`.
4. Выберите ветку (например `main`) и папку `/ (root)`, нажмите **Save**.
5. Через ~1 минуту сайт будет доступен по адресу
   `https://<ваш-логин>.github.io/<репозиторий>/`.

Ничего собирать не нужно — это статические файлы.

## Подключение реальной карты (RustMaps API) — на будущее

В `script.js`, в функции `initMap()`, есть блок-заглушка с псевдокодом:
пользователь вводит **seed + размер** карты, запрос к RustMaps возвращает изображение
карты и координаты монументов, которые накладываются на 3D-плоскость. Нужен свой
API-ключ RustMaps. Сейчас реализована процедурная (демо) заглушка.

## Дизайн-система

Сгенерирована скиллом `ui-ux-pro-max` и сохранена в `design-system/rust-wipe-hub/MASTER.md`.
Авто-рекомендация выдавала неоново-фиолетовую палитру и паттерн «рассылка» — они заменены
на палитру **«Operation orange on dark»** (из БД скилла) и паттерн «content-dense tabbed
tool hub», соответствующие брифу. Все цвета/отступы/тени вынесены в CSS-переменные —
в разметке «сырых» hex нет.
